BI & Dashboard Case Study

NYFS Digital Dashboard

A live public-data intelligence product that transforms NYC restaurant inspection records into a more usable dashboard for comparing safety patterns, locations, cuisine categories, and risk signals.

GitHub Live Dashboard

Business Question

Where are food-safety risks concentrated, and which inspection patterns should decision-makers monitor?

Analytical Value

Turns hard-to-use public health records into KPIs, maps, and comparison views for faster inspection and location analysis.

What This Demonstrates

  • Cleaning and structuring messy public inspection data.
  • Engineering dashboard-ready fields such as risk level, latest-inspection snapshot, and violation categories.
  • Designing an interactive product experience for practical restaurant comparison.
  • Connecting analytics output to an audience: diners, students, tourists, and public-data users.

Employer Takeaway

This project shows the full business analyst workflow: define the user problem, prepare messy public data, expose the evidence through a web dashboard, and make the result easy for a non-technical visitor to inspect.

Portfolio Positioning

This is the clearest data-product example in the portfolio because it combines data processing, dashboard design, user interaction, and business-style decision support.
